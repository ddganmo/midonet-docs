Author: Jan Hilberath <jan@midokura.com>
Date:   Sun Jan 17 18:14:35 2016 +0900

    Remove MidoNet CLI based BGP port creation and binding
    
    With the new Edge Router model in MidoNet v5.0 it's not necessary
    to create and bind the BGP ports within MidoNet CLI any more:
    
     midonet> router router0 add port address 198.51.100.2 net 198.51.100.0/30
     midonet> host host0 add binding port router0:port0 interface eth1
    
    Instead, the ports are created and bound via Neutron:
    
     neutron port-create ... \
       --binding:host_id ... \
       --binding:profile type=dict interface_name=... \
       --fixed-ip ip_address=...
    
    This patch removes the MidoNet CLI parts from the BGP Uplink
    Configuration section and adds a link to the "Edge Router Setup"
    section, which describes the port creation via Neutron.
